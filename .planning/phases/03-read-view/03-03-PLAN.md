  <name>Task 1: Add SearchModel and AttachmentModel CXX-Qt QObjects</name>
  <files>src/cxxqt.rs, build.rs</files>
  <action>
    Per READ-07, ATCH-01, ATCH-02, ATCH-03:

    Add two new QObjects to src/cxxqt.rs following the existing CXX-Qt 0.8 pattern:

    **1. SearchModel** (READ-07 — search emails):
    - Properties: `#[qproperty(i32, result_count)]`, `#[qproperty(QString, search_query)]`
    - Internal: `results: Vec<EmailMetadata>` cache
    - Invokables:
      - `search(account_id: &QString, query: &QString)` — calls `STORAGE.lock().search_emails(qstring_to_string(account_id), qstring_to_string(query))`, updates result_count and internal cache
      - `get_result_id(index: i32) -> QString`
      - `get_result_subject(index: i32) -> QString`
      - `get_result_from(index: i32) -> QString`
      - `get_result_date(index: i32) -> QString`
      - `get_result_read(index: i32) -> bool`
      - `clear_search()` — resets query and results

    **2. AttachmentModel** (ATCH-01, ATCH-02, ATCH-03 — attachment list, download, PDF view):
    - Properties: `#[qproperty(i32, attachment_count)]`, `#[qproperty(QString, current_email_id)]`, `#[qproperty(QString, download_status)]`
    - Internal: `attachments: Vec<AttachmentMetadata>` cache
    - Invokables:
      - `load_attachments(email_id: &QString)` — calls `STORAGE.lock().list_attachments(qstring_to_string(email_id))`, updates attachment_count and cache (ATCH-01)
      - `get_attachment_filename(index: i32) -> QString` — for list display (ATCH-01)
      - `get_attachment_content_type(index: i32) -> QString` — to determine viewer (ATCH-01, ATCH-03)
      - `get_attachment_size(index: i32) -> i64` — size in bytes (ATCH-01)
      - `get_attachment_size_text(index: i32) -> QString` — human-readable size like "2.3 MB" (ATCH-01)
      - `is_attachment_downloaded(index: i32) -> bool` — check if already on disk (ATCH-02)
      - `get_attachment_local_path(index: i32) -> QString` — path for PDF viewer (ATCH-03)
      - `download_attachment(index: i32) -> bool` — downloads attachment to device (ATCH-02):
        a. Get attachment metadata from cache at index
        b. Connect to IMAP (get account config from STORAGE, call `imap_conn::connect_imap()`)
        c. Fetch the active email's UID from Storage via `get_email(email_id)` → get `uid` and `folder`
        d. Select folder, call `imap_conn::fetch_attachment_part(session, uid, &part_number)`
        e. Save bytes to `dirs::data_dir()/remailable/attachments/{account_id}/{attachment_id}/{filename}`
        f. Update `STORAGE.lock().mark_downloaded(attachment_id, &local_path)`
        g. Refresh attachments list
        h. Return true on success
        NOTE: This is synchronous on main thread (same as sync_all in Phase 2). For v1, this is acceptable.

    **3. Update build.rs** — add new QML files:
    ```rust
    .qml_file("qml/SearchBar.qml")
    .qml_file("qml/AttachmentList.qml")
    .qml_file("qml/PdfViewer.qml")
    ```

    Create minimal QML stubs for SearchBar.qml, AttachmentList.qml, PdfViewer.qml so build.rs compiles (full implementation in Task 2).

    **Implementation notes:**
    - `get_attachment_size_text` implementation: if size >= 1_048_576 → format!("{:.1} MB", size as f64 / 1_048_576.0), if >= 1024 → format!("{:.1} KB", size as f64 / 1024.0), else → format!("{} B", size)
    - download_attachment needs the account_id from the attachment to connect to the right IMAP server. Get it from `STORAGE.lock().get_attachment_by_id()` or from cache.
    - For the IMAP download, need the email's UID and folder. Get from `STORAGE.lock().get_email(email_id)`.
