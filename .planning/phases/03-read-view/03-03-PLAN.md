    **2. Create AttachmentListModel**

    In the qobject module, add the declaration:
    ```rust
    extern "RustQt" {
        #[qobject]
        #[qml_element]
        #[qproperty(i32, attachment_count)]
        type AttachmentListModel = super::AttachmentListModelRust;
    }

    extern "RustQt" {
        #[qinvokable]
        fn load_attachments(self: Pin<&mut AttachmentListModel>, email_id: &QString);

        #[qinvokable]
        fn get_attachment_filename(self: &AttachmentListModel, index: i32) -> QString;

        #[qinvokable]
        fn get_attachment_size(self: &AttachmentListModel, index: i32) -> i32;

        #[qinvokable]
        fn get_attachment_content_type(self: &AttachmentListModel, index: i32) -> QString;

        #[qinvokable]
        fn download_attachment(self: Pin<&mut AttachmentListModel>, index: i32) -> QString;

        #[qinvokable]
        fn is_attachment_downloaded(self: &AttachmentListModel, index: i32) -> bool;

        #[qinvokable]
        fn get_attachment_download_path(self: &AttachmentListModel, index: i32) -> QString;
    }
    ```

    Rust backing struct:
    ```rust
    #[derive(Default)]
    pub struct AttachmentListModelRust {
        attachment_count: i32,
        attachments: Vec<AttachmentMetadata>,
    }
    ```

    Implementation:
    - `load_attachments`: Lock STORAGE, call list_attachments(email_id), store in self.attachments, update count
    - `get_attachment_filename/size/content_type`: Return field at index from self.attachments (bounds check, return ""/0/""/"application/octet-stream" for out of bounds)
    - `download_attachment`: Look up the attachment at index, get its stored path from imap sync. If the file doesn't exist locally, return empty string. Otherwise, copy it to `dirs::data_dir()/remailable/downloads/{email_id}/{filename}`, call STORAGE.mark_downloaded(id, download_path), return the download path as QString.
    - `is_attachment_downloaded`: Check if attachments[index].download_path.is_some()
    - `get_attachment_download_path`: Return the download_path if Some, empty string if None

    For download_attachment, the attachment files were saved by the sync engine to `attachments/{account_id}/{email_id}/{filename}` relative to the data directory. Construct the full source path using `dirs::data_dir()/remailable/attachments/{account_id}/{email_id}/{filename}`. Copy to `dirs::data_dir()/remailable/downloads/{email_id}/{filename}`.
