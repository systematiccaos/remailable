<task type="auto">
  <name>Task 1: Create CXX-Qt bridge models for folders, email list, and email content</name>
  <files>src/cxxqt.rs, src/main.rs</files>
  <action>
    Extend src/cxxqt.rs with three new QObjects following the established CXX-Qt 0.8 patterns. Use QString backing fields for qproperty, use the `STORAGE` global for data access, and use the established `qstring_to_string`/`string_to_qstring` helpers.

    **1. FolderListModel** — Provides folder list for the active account.

    Rust backing struct:
    ```rust
    #[derive(Default)]
    pub struct FolderListModelRust {
        folder_count: i32,
        folders: Vec<String>,  // internal cache of folder names
    }
    ```

    qproperties: `folder_count` (i32)

    qinvokables:
    - `refresh_folders(self: Pin<&mut FolderListModel>)` — load folders for active account from Storage, update folder_count
    - `get_folder_name(self: &FolderListModel, index: i32) -> QString` — return folder name at index
    - `get_folder_email_count(self: &FolderListModel, index: i32) -> i32` — return email count for folder (query Storage::list_emails_by_folder().len())
    - `select_folder(self: Pin<&mut FolderListModel>, index: i32)` — set current folder on AppModel, trigger email list refresh

    Implementation: On refresh_folders, lock STORAGE, get active_account_id from AppModel, call STORAGE.list_folders(&account_id). Store results in self.folders. For get_folder_email_count, call STORAGE.list_emails_by_folder(&account_id, &folder_name) and return count as i32.

    **2. EmailListModel** — Provides email list for the selected folder.

    Rust backing struct:
    ```rust
    #[derive(Default)]
    pub struct EmailListModelRust {
        email_count: i32,
        emails: Vec<EmailMetadata>,  // internal cache
    }
    ```

    qproperties: `email_count` (i32)

    qinvokables:
    - `refresh_emails(self: Pin<&mut EmailListModel>)` — load emails for active account + selected folder from Storage
    - `get_email_subject(self: &EmailListModel, index: i32) -> QString`
    - `get_email_from(self: &EmailListModel, index: i32) -> QString`
    - `get_email_date(self: &EmailListModel, index: i32) -> QString`
    - `get_email_id(self: &EmailListModel, index: i32) -> QString`
    - `get_email_read(self: &EmailListModel, index: i32) -> bool` — returns true if email is read
    - `get_email_has_attachment(self: &EmailListModel, index: i32) -> bool` — check if email has attachments in DB
    - `toggle_email_read(self: Pin<&mut EmailListModel>, index: i32)` — toggle read/unread in DB and on IMAP server
    - `select_email(self: Pin<&mut EmailListModel>, index: i32)` — set selected_email_id on AppModel, mark as read if unread

    Implementation: refresh_emails locks STORAGE, gets account_id from AppModel (via the global), gets selected folder name, calls STORAGE.list_emails_by_folder. For toggle_email_read, locks STORAGE, calls mark_email_read, then connects to IMAP with connect_imap to call mark_email_seen (handle connection failure gracefully — log error but don't block UI). For get_email_has_attachment, call STORAGE.list_attachments(&email_id) and check if non-empty.

    IMPORTANT: The IMAP mark_email_seen call requires obtaining an IMAP session. Use imap_conn::connect_imap to get a session for the current account (get account config from STORAGE to retrieve IMAP credentials). This is synchronous and blocks the UI — acceptable for v1, same as the synchronous validation pattern established in Phase 2. If the IMAP connection fails, log the error but still update the local DB.

    **3. EmailContentModel** — Provides the content of a single email for viewing.

    Rust backing struct:
    ```rust
    #[derive(Default)]
    pub struct EmailContentModelRust {
        email_subject: cxx_qt_lib::QString,
        email_from: cxx_qt_lib::QString,
        email_date: cxx_qt_lib::QString,
        email_body: cxx_qt_lib::QString,
        email_format: cxx_qt_lib::QString,  // "plain" or "html"
        has_html: bool,
    }
    ```

    qproperties: `email_subject`, `email_from`, `email_date`, `email_body`, `email_format` (all QString), `has_html` (bool)

    qinvokables:
    - `load_email(self: Pin<&mut EmailContentModel>, email_id: &QString)` — load email metadata and body from Storage, set all properties
    - `show_html(self: Pin<&mut EmailContentModel>)` — switch email_body to HTML content and email_format to "html"
    - `show_plain_text(self: Pin<&mut EmailContentModel>)` — switch email_body to plain text and email_format to "plain"

    Implementation: load_email locks STORAGE, calls get_email_text_body to read the body file, sets email_subject/from/date/format. If HTML body exists, set has_html = true. show_html loads the HTML body file, show_plain_text loads the text body file.

    **4. Extend AppModel** — Add navigation properties.


    In the qobject bridge block, add qproperties for AppModel:
    - `#[qproperty(QString, selected_folder)]`
    - `#[qproperty(QString, selected_email_id)]`

    **5. Update main.rs** — ensure email_parse module is declared: `pub mod email_parse;` (should be there from Plan 01).

    Ensure build compiles with `cargo check`.
