  <files>src/email_parse.rs, Cargo.toml</files>
  <behavior>

    - Test 2: parse_email() extracts HTML body from a multipart MIME message
    - Test 3: parse_email() extracts attachment metadata (filename, content_type, size) from MIME
    - Test 4: sanitize_html_for_eink() strips script/style tags and external resources
    - Test 5: sanitize_html_for_eink() preserves safe formatting tags (p, b, i, a, br, hr, ul, ol, li, h1-h6, blockquote, table)
    - Test 6: parse_email() handles plain-text-only emails (no HTML part) gracefully
    - Test 7: parse_email() handles email with no attachments gracefully
    - Test 8: Thread key normalization strips Re:/Fwd:/Fw: prefixes and normalizes case
  </behavior>
  <action>
    Add dependencies to Cargo.toml:
    - `mail-parser = "0.9"` (pure Rust MIME parsing, no C deps, cross-compiles well)
    - `ammonia = "4"` (HTML sanitization, whitelist-based, pure Rust)

    Create `src/email_parse.rs` with the following:

    1. **ParsedEmail struct**: holds text_body (Option<String>), html_body (Option<String>), attachments (Vec<AttachmentInfo>), date, subject, from_addr.

    2. **AttachmentInfo struct**: filename (String), content_type (String), size (usize), content_id (Option<String>), content (Vec<u8>).

    3. **parse_email(raw_bytes: &[u8]) -> Result<ParsedEmail, String>**: Use mail_parser to parse the raw MIME message. Extract:
       - text/plain body part → text_body
       - text/html body part → html_body
       - All non-text attachments → AttachmentInfo with filename, content_type, size (content.len()), content_id, and the raw content bytes
       - Subject and From from the envelope
       For emails with only text/plain, html_body = None. For emails with only text/html, text_body = None.

    4. **sanitize_html_for_eink(html: &str) -> String**: Use ammonia to sanitize HTML for e-ink display:
       - Strip script, style, iframe, object, embed, form tags
       - Strip img tags with external src (keep alt text as text)
       - Preserve safe tags: p, br, hr, b, i, em, strong, u, a (href only), ul, ol, li, h1-h6, blockquote, pre, code, table, tr, td, th, span, div
       - Strip all CSS inline styles (font sizes, colors — let QML handle rendering)
       - Strip class attributes
       - Convert relative hrefs to absolute where possible
       - Return sanitized HTML string suitable for QML Text element with textFormat: Text.RichText

    5. **normalize_subject_for_thread(subject: &str) -> String**: Strip "Re:", "Fwd:", "Fw:", "RE:", "FWD:" prefixes (with optional brackets like [Fwd:]), convert to lowercase, trim whitespace. Used for thread grouping by subject.

    6. **extract_text_fallback(html: &str) -> String**: Simple HTML-to-text fallback for when ammonia is too aggressive or for very complex emails. Strip all HTML tags, preserve paragraph breaks. Use html2text-like logic (iterate chars, replace </p> and <br> with newlines).

    Add `pub mod email_parse;` to src/main.rs.

    Write unit tests in src/email_parse.rs covering all 8 behaviors above. Use sample MIME messages constructed in test code (multipart with text/html attachments).
  </action>

  </verify>

</task>

       ```sql

           filename TEXT NOT NULL,
       ```
